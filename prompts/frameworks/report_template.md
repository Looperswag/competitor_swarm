# 竞品分析报告模板框架 v3.0

> 本文档定义了 CompetitorSwarm 系统生成的竞品分析报告的标准结构和格式要求。

## 报告结构标准

### 1. 元信息（必须）
```markdown
- **分析目标**: [竞品名称]
- **生成时间**: [时间戳]
- **分析耗时**: [秒数]
- **状态**: ✓ 成功 / ✗ 失败
- **对比产品**: [对比产品列表，如有]
- **发现数量**: [总条数]
```

### 2. 执行摘要（必须，3分钟速读）
```markdown
## 一句话定调
[用1句话概括竞品的核心定位和我方的核心差距，不超过50字]

## 核心发现速览
- [数据1] + [结论1]（置信度：高/中/低）
- [数据2] + [结论2]（置信度：高/中/低）
- [数据3] + [结论3]（置信度：高/中/低）

## 关键行动项（Top 3）
| 优先级 | 行动项 | 时间线 | 预期ROI |
|--------|--------|--------|---------|
| P0 | [行动1] | 短期/中期/长期 | 高/中/低 |
```

### 3. 竞品定位画像（必须）
```markdown
## 竞品定位画像

[100字以内，描述竞品的核心定位、目标用户、核心价值主张]
- **核心定位**：[一句话定义]
- **目标用户**：[用户画像]
- **价值主张**：[核心卖点]
```

### 4. 核心维度对比表（必须）
```markdown
## 核心维度对比表

| 维度 | 竞品表现 | 我方差距 | 战略含义 | 数据支撑 |
|------|---------|---------|---------|---------|
| 产品功能 | [量化评分 + 关键数据] | 领先/持平/落后 | [一句话] | [数据来源] |
| 用户体验 | [量化评分 + 关键数据] | 领先/持平/落后 | [一句话] | [数据来源] |
| 技术能力 | [量化评分 + 关键数据] | 领先/持平/落后 | [一句话] | [数据来源] |
| 市场份额 | [量化评分 + 关键数据] | 领先/持平/落后 | [一句话] | [数据来源] |
| 商业化 | [量化评分 + 关键数据] | 领先/持平/落后 | [一句话] | [数据来源] |
```

### 5. SWOT 分析（必须，结构化表格）
```markdown
## SWOT 分析

### 优势 (Strengths)
| 优势项 | 量化证据 | 可持续性 | 来源于 |
|-------|---------|---------|--------|
| [优势1] | [数据/事实] | 高/中/低 | [Agent名] |

### 劣势 (Weaknesses)
| 劣势项 | 影响程度 | 紧迫性 | 来源于 |
|-------|---------|--------|--------|
| [劣势1] | 高/中/低 | 高/中/低 | [Agent名] |

### 机会 (Opportunities)
| 机会项 | 市场规模(估) | 我方匹配度 | 捕获难度 |
|-------|-------------|-----------|---------|
| [机会1] | [数字或范围] | 高/中/低 | 高/中/低 |

### 威胁 (Threats)
| 威胁项 | 发生概率 | 影响程度 | 应对策略概要 |
|-------|---------|---------|-------------|
| [威胁1] | 高/中/低 | 高/中/低 | [一句话应对] |
```

### 6. 阶段策略摘要（必须）
```markdown
## 阶段策略摘要

### Phase 1 信息收集
- [收集到的关键信息摘要]

### Phase 2 交叉验证
- 验证通过: X 条
- 过滤淘汰: Y 条

### Phase 3 红蓝辩论
- 辩论轮数: N
- 红队观点数: X
- 蓝队观点数: Y
```

### 7. 各维度分析详情（必须）
```markdown
## 侦察分析
[Scout Agent 的详细发现]

## 体验分析
[Experience Agent 的详细发现]

## 技术分析
[Technical Agent 的详细发现]

## 市场分析
[Market Agent 的详细发现]
```

### 8. 红蓝对抗涌现结论（必须，表格化）
```markdown
## 红蓝对抗涌现结论

| 争议点 | 红队批判 | 蓝队辩护 | 涌现结论 | 置信度 |
|-------|---------|---------|---------|--------|
| [争议1] | [观点] | [观点] | [平衡结论] | 高/中/低 |
```

### 9. 战略定位矩阵（必须）
```markdown
## 战略定位矩阵

| 维度 | 竞品表现 | 我方差距 | 战略含义 |
|------|---------|---------|---------|
| [维度1] | 强/中/弱 | 领先/持平/落后 | [一句话] |
```

### 10. 风险/机会矩阵（必须）
```markdown
## 风险/机会矩阵

| 类型 | 事项 | 影响程度 | 发生概率 | 应对策略 |
|------|------|---------|---------|---------|
| 风险 | [具体风险] | 高/中/低 | 高/中/低 | [一句话应对] |
| 机会 | [具体机会] | 高/中/低 | 高/中/低 | [一句话行动] |
```

### 11. 可执行建议（必须）
```markdown
## 可执行建议

### 建议 1：[标题]

| 属性 | 值 |
|------|-----|
| 优先级 | 高/中/低 |
| 实施难度 | 简单/中等/困难 |
| 预期 ROI | 高/中/低 |
| 时间线 | 短期(1-3月)/中期(3-6月)/长期(6-12月) |

**行动描述**：[2-3 句话描述具体行动]

**实施步骤**：
1. [步骤 1]
2. [步骤 2]

**成功指标**：[可量化的 KPI]
```

### 12. 附录（可选）
```markdown
## 附录

### 分析方法论
- Swarm 协作模式
- Stigmergy 通信机制
- 红蓝对抗涌现

### 数据来源索引
- [来源1]: [URL]
- [来源2]: [URL]

### 洞察追溯
- [洞察1]: 来源于 [agent1, agent2]
```

## 数据驱动要求

### 量化数据要求
- **最少5个量化数据**：每份报告应包含至少5个带数字的论据
- **数据格式**：数字 + 单位 + 时间范围 + 来源
- **对比量化**：给出"我们 vs 竞品"的量化对比

### 数据示例
| 差的描述 | 好的描述 |
|---------|---------|
| 用户体验好 | Task Completion Rate 92%，高于行业平均15% |
| 市场份额大 | 市场份额约23%，位居品类第二 |
| 增长快 | 月活用户同比增长45%，达500万 |
| 技术强 | API响应时间 P99 < 200ms |
| 用户粘性强 | DAU/MAU 比率达35%，高于行业平均20% |

## 报告质量检查清单

### 结构完整性
- [ ] 元信息是否完整？
- [ ] 执行摘要是否在3分钟内可读完？
- [ ] 是否有竞品定位画像？
- [ ] 核心维度对比表是否覆盖4个以上维度？
- [ ] SWOT表格是否每项至少有2条？
- [ ] 红蓝对抗涌现结论是否表格化？

### 数据驱动
- [ ] 是否有至少5个量化数据？
- [ ] 数据是否标注时间范围？
- [ ] 数据是否标注来源？
- [ ] 是否有"我们 vs 竞品"的量化对比？

### Swarm 特色保留
- [ ] 是否保留红蓝对抗涌现？
- [ ] 是否有置信度标注？
- [ ] 是否有 Agent 来源追溯？
- [ ] 是否有证据链？

## 报告输出格式

### Markdown 格式
- 使用标准 Markdown 语法
- 表格使用 GFM (GitHub Flavored Markdown) 格式
- 标题层级：H1 > H2 > H3 > H4

### HTML 格式
- 使用报告模板生成可视化 HTML
- 表格需要 CSS 样式美化
- 支持打印和导出 PDF
